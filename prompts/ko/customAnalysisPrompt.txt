  ═══════════════════════════════════

  형식: hold:plan|작성 일시|내용(기한)
  예시: hold:plan|1988/09/29 21:00|내일 아침 9시에 기상 후 쇼핑을 가고 교복 착용을 엄금한다(1988/09/30 09:00)

  【소거 조건 — 하나라도 만족하면 발동】
    ✦ 실행됨: 분석 대상 텍스트에서 해당 계획이 실제로 실행되었다고 명시됨
    ✦ 취소됨: 분석 대상 텍스트에서 해당 계획이 포기되었다고 명시됨
    ✦ 기한 경과: 현재 시간이 해당 계획의 기한을 이미 지남(본문에 직접 언급이 없어도 만료로 간주)
    → 반드시 hold-: 로 소거한다!

  【병합 갱신 및 중복 방지】hold:plan 을 만들기 전에 기존 보류 장부와 반드시 비교한다.
    1. 단순 재언급: 계획 자체에 변화가 없으면 재출력 금지
    2. 세부 갱신: 같은 기존 계획에 새 세부가 추가되면 → 먼저 hold-: 로 구항목을 삭제한 뒤, 신구 세부를 모두 합친 최종 항목 1건만 hold: 로 다시 쓴다. 구버전과 신버전 공존 금지!
    3. 일괄 병합: 같은 턴에서 생긴 관련 계획은 반드시 한 줄로 합친다. 여러 hold: 줄로 나누는 것은 절대 금지!

  【유형 2: suspense】주인공이 능동적으로 통제하지 못하는 미해결 사항
  ═══════════════════════════════════

  예를 들면 다음과 같지만 여기에 한정되지 않는다.
    • 외부 위협, 경고, 카운트다운("사흘 뒤에 다시 보자")
    • 미해결 수수께끼나 이상 단서(책상 위에 낯선 쪽지가 나타남)
    • 정보 비대칭(누군가가 뭔가를 알지만 말하지 않음)
    • 타인이 주인공에게 한 약속, 또는 주인공이 타인에게 진 빚이나 의무
    • 복선, 전조, 불길한 조짐

  형식: hold:suspense|기록 일시|내용
  예시:
    hold:suspense|1988/09/29 23:30|검은 로브의 인물이 "사흘 뒤에 다시 보자"라고 경고했지만 정체는 불명
    hold:suspense|1988/09/29 23:25|방 책상 위에 낯선 필체의 쪽지가 나타났고, 글씨는 의도적으로 위장되어 있다
    hold:suspense|1988/09/27 14:20|Anna는 {{user}}와 Bella가 함께 있는 것을 보고도 말하지 않았지만 속으로 의심했다
    hold:suspense|1988/09/29 22:00|주점 주인이 다음번에 {{user}}를 위해 좋은 술 한 통을 남겨 두겠다고 약속했다

  🔴 시간만 쓰고 날짜를 빼먹지 말고, 날짜만 쓰고 시간을 빼먹지도 마라.
   올바름: hold:suspense|1988/09/29 14:21|방 책상 위에 낯선 필체의 쪽지가 나타났다
   잘못: hold:suspense|14:21|방 책상 위에 낯선 필체의 쪽지가 나타났다 (날짜 누락)
   잘못: hold:suspense|1988/09/29|방 책상 위에 낯선 필체의 쪽지가 나타났다 (시간 누락)

  🔴【suspense 는 시간 경과만으로 자동 소거하면 안 된다!】
  suspense 에는 기한 개념이 없다. 미해결 사항은 "시간이 오래 지났다"는 이유만으로 사라지지 않는다.

  【소거 조건 — 회고적 소거만 허용】
  분석할 때마다 기존 suspense 항목을 하나씩 확인하고, 오직 이 질문만 한다.

    "이 suspense 항목이 설명하는 일이 분석 대상 텍스트 안에서
     이미 해결되었는가, 밝혀졌는가, 뒤집혔는가, 혹은 완전히 불가능해졌는가?"

    ✦ 예, 그리고 명확한 본문 근거가 있음 → hold-: 로 소거
    ✦ 아니오 → 그대로 유지
    ✦ 불확실함 → 그대로 유지

  🔴 "이쯤이면 벌어졌어야 한다"거나 "너무 오래 끌었다"는 감각만으로 suspense 를 소거하지 마라!
     이미 벌어진 일만 소거할 수 있다. 잘못 지우는 것보다 남겨 두는 편이 낫다.

  【suspense 의 병합 갱신】같은 원칙을 적용한다.
    ✦ 같은 suspense 에 새 진전이 생겼지만 아직 완전히 해결되지 않음 → hold-: 로 구항목을 지우고, 갱신된 hold:suspense 를 작성
    ✦ 여러 suspense 가 사실 같은 일임이 드러남 → hold-: 로 구항목을 모두 지우고, 통합된 hold:suspense 1건만 작성

【event 요약】
· 120~240자 정도를 기준으로, 누가 누구에게 무엇을 했는지, 무엇을 말했는지, 원문에 명시된 감정이나 관계 변화가 있다면 무엇인지, 어떤 단서나 결과가 생겼는지 작성.
· 시점은 【차가운 감시 카메라】 + 【경찰 진술서】로 고정한다. 시각적으로 보이는 동작, 청각적으로 들리는 대사, 본문에 명시된 사실만 쓴다. 문학적 수식은 금지한다.
· 추상적인 말은 금지.
  ❌ 「{{user}}와 민준은 대화했고 관계가 조금 좋아졌다.」
  ✅ 「{{user}}는 폐역에서 민준에게 실종된 여동생의 수첩을 보여 주었고, 민준은 의심을 거두지 않았지만 수첩의 암호가 구시가지 창고 번호를 가리킨다고 인정했다. 민준은 {{user}}를 단순 의뢰인이 아니라 협력자로 대하기 시작했다.」
· 원문에 명시되지 않은 내용의 날조 금지. 본문이 객관적 동작만 적었다면 요약도 동작만 적고, 숨은 속내나 깊은 동기를 추측하지 않는다.
· 분위기 총평 금지. "생활감이 묻어난다" "분위기가 바뀌었다" 같은 주관적 감상을 쓰지 않는다.
· 이야기 이어쓰기 금지: event 서술은 【분석 대상 텍스트】에 나온 마지막 명시적 동작/대사에서 반드시 멈춘다.
  원문에 없는 후속 동작, 반응, 퇴장, 마무리를 보태지 않는다. 논리적으로 "당연히 이어질" 내용이어도 금지.
  ❌ 오답 예시(원문이 "여주가 밀크티를 원한다"에서 끝남):
     "…남주가 무엇을 마실지 묻고, 여주가 밀크티라고 답하자 남주가 고개를 끄덕이며 사러 간다."
     ("고개를 끄덕이며 사러 간다"는 원문에 없는 이어쓰기)
  ✅ 정답 예시:
     "…남주가 무엇을 마실지 묻고, 여주가 밀크티라고 답한다."
     (원문 마지막 문장에서 멈추고, 후속을 보태지 않음)

【event 필드의 유일성과 위치】
- 매 응답에는 event 항목을 정확히 1개만 작성한다.
- 같은 층에 여러 사건 조각(예: 대화→전투→헤어짐)이 있어도, 120~240자 요약 1개로 시간 순서대로 통합하고 여러 event 줄로 분할하지 않는다.
- event 는 <horaeevent> 안에만 작성하고, <horae> 안에는 event: 줄을 쓰지 않는다.
- 기록할 만한 진행이 정말 없는 경우(매우 드묾)에도 event 1개는 반드시 출력하고, 중요도는 normal 로 하며 가장 핵심적인 객관 동작을 적는다.
